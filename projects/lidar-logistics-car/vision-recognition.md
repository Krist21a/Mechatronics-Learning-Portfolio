# Vision Recognition

## 目标

记录智能仓储搬运机器人目标识别模块的设计思路、YOLOv11 检测尝试、相机标定、点云识别方案和最终调试结论。

## Design Goal

目标识别模块需要为机械臂抓取提供物体位置。理想流程如下：

```text
camera / point cloud sensor
    -> object detection
    -> object position estimation
    -> coordinate transformation
    -> arm target coordinate
    -> grasp action
```

在项目调试中，主要对比了 YOLOv11 图像识别和点云识别两种方案。

## YOLOv11 Detection

### 实现思路

- 使用开源数据集和自建数据集合并训练。
- 使用 Mosaic 数据增强提高样本多样性。
- 使用自适应锚框计算等策略提升模型性能。
- 尝试模型轻量化，并部署到树莓派。
- 识别目标为箱体 / 货物类物体。

### 调试现象

YOLO 模型可以输出目标框和置信度，例如箱体目标识别结果中出现过 `Box 0.93` 的检测结果。但在实际环境中，识别效果容易受到背景、光照、目标纹理和训练集规模影响。

### 问题分析

- 数据集偏小，场景覆盖不足。
- 前期训练轮次和参数仍需优化。
- 背景信息容易干扰目标框稳定性。
- 检测框只能提供图像平面信息，还需要额外标定和深度信息才能稳定转为抓取坐标。

## Camera Calibration

相机标定部分用于建立图像坐标与真实空间坐标之间的关系。

已整理内容：

- 测定相机内参矩阵。
- 测定畸变系数。
- 进行手眼标定。
- 建立像素坐标与物体世界坐标之间的坐标变换矩阵。

后续需要补充标定板图片、标定脚本、误差数据和坐标系示意图。

## Point Cloud Recognition

### 实现思路

点云识别方案通过几何信息提取目标，不依赖目标纹理和颜色特征。

主要步骤：

1. 获取点云数据。
2. 使用 RANSAC 算法进行平面分割。
3. 调整阈值，提取非地面 / 非桌面障碍物点云。
4. 执行欧氏聚类。
5. 分析聚类特征，提取目标物体。
6. 输出目标空间位置，供机械臂抓取使用。

### 优点

- 受环境光照影响小。
- 受目标纹理和背景颜色影响小。
- 在定位精度上比纯 2D 目标框更直接。
- 更适合为机械臂提供空间坐标。

## Comparison

| 方案 | 优点 | 问题 | 当前结论 |
| --- | --- | --- | --- |
| YOLOv11 | 检测直观，可以识别箱体类别 | 受背景、光照、数据集和训练参数影响；空间定位还需要额外转换 | 可作为识别尝试和后续优化方向 |
| 点云识别 | 受光照和纹理影响小，定位更直接 | 需要调阈值和聚类参数 | 当前实际效果更稳定，最终采用 |

## Final Decision

通过比较输出坐标与实际物体坐标，点云识别在当前工作环境中的实际效果明显优于 YOLO 模型。因此项目最终采用点云识别作为主要目标定位方案。

这个结论也说明：在工程项目中，算法选择不能只看模型先进程度，还要看传感器条件、部署平台、数据质量和任务需求。对于机械臂抓取任务，稳定输出空间坐标比单纯检测框更重要。

## Future Work

- 整理相机标定与手眼标定过程。
- 将点云目标坐标与机械臂控制坐标系打通。
- 增加抓取成功率测试表。
