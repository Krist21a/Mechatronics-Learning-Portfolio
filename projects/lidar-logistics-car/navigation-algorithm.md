# Navigation Algorithm

## 目标

记录智能仓储搬运机器人的建图、定位、路径规划、局部避障和导航调试过程。当前重点是基于 ROS2 的二维激光建图与室内自主导航。

## System Flow

```text
RPLIDAR A1M8      -> /scan
wheel encoder     -> /wheel/odom
MPU6050 IMU       -> /imu/data_raw

/scan + /wheel/odom + /imu/data_raw
        -> time sync / TF
        -> robot_localization EKF
        -> /odometry/filtered
        -> slam_toolbox
        -> /map
        -> RViz2 visualization
        -> AMCL + global planner + DWB
        -> chassis velocity command
```

## Mapping

建图流程使用 RPLIDAR A1M8、轮式编码器、MPU6050 IMU 和 ROS2 Humble 环境。

### 关键工作

1. 完成树莓派远程桌面部署和 ROS2 环境搭建。
2. 建立 LiDAR、编码器、IMU 与底盘串口通信链路。
3. 构建 TF 树和多源传感器融合框架。
4. 使用 `slam_toolbox` 实现室内二维激光建图。
5. 使用 RViz2 和 NoMachine 查看建图过程与地图结果。

### 关键硬件与软件

| 主控平台 | Raspberry Pi 4B, Ubuntu 22.04, ROS2 Humble |

## Map Correction and Virtual Walls

原始建图结果在玻璃、空隙和门口区域存在误判。主要表现为：

- 玻璃走廊 / 空隙区域导致激光穿透或误判，路径规划可能误入不可通行区域。
- 可通行区域与实际障碍不一致。
- 局部边界存在虚假开放区域，影响全局路径规划。

### 修正策略

基于 RViz 采点和 PGM 栅格编辑构建虚拟墙：

1. 使用 RViz 的 Publish Point 获取关键端点坐标。
2. 根据折线 / 直线几何约束生成虚拟墙。
3. 写回 PGM 地图并重新生成 YAML 地图文件。
4. 在 RViz2 中检查修正后地图和虚拟墙标注结果。

### 修正效果

- 提高规划地图与真实环境的一致性。
- 将“传感器可见环境”修正为“机器人真实可通行环境”。
- 增强导航安全性和可靠性。
- 为后续稳定导航提供更可靠的地图基础。

## Navigation Pipeline

导航实验基于修正后的地图进行：

| 模块 | 作用 |
| --- | --- |
| AMCL | 完成定位初始化与位姿更新 |
| Global Planner | 基于地图生成全局路径 |
| Costmap | 维护障碍物和膨胀区域 |
| DWB Controller | 完成局部轨迹跟踪和避障 |
| RViz2 | 可视化起点、目标点、规划路径和机器人状态 |

### 导航过程要点

- 基于修正地图进行全局路径规划。
- 使用 AMCL 完成定位初始化与位姿更新。
- 使用 DWB 控制器完成局部轨迹跟踪。
- 机器人完成门口通行与长距离自主导航实验。
- RViz2 中可观察起点位置、目标点位置、全局规划路径、虚拟墙 / 门口约束区域和局部避障更新区域。

## Chassis Kinematics

底盘采用四麦克纳姆轮结构，需要完成机器人速度与四个轮子速度之间的转换。

### 逆运动学

逆运动学用于根据机器人期望速度计算四个麦克纳姆轮的线速度。

输入：

- 机器人期望前后速度。
- 机器人期望横向速度。
- 机器人期望角速度。
- 机器人半长、半宽和轮半径等结构参数。

输出：

- 四个麦克纳姆轮的目标线速度 / 角速度。

### 正运动学

正运动学用于根据四个轮子的实际线速度反算机器人当前运动速度。

用途：

- 估计机器人底盘运动状态。
- 为里程计和导航系统提供运动反馈。
- 检查轮速控制与实际运动是否一致。

## Experiment Result

已完成的功能：

- 二维建图。
- 地图修正。
- 定位初始化。
- 全局路径规划。
- 局部避障。
- 目标点导航。

实验现象：

- 在修正地图后，导航路径与真实可通行区域更一致。
- 机器人可以完成短程与长距离目标点到达。
- 门口、狭窄通道和玻璃区域仍然是影响导航稳定性的重点场景。

## Problems and Tuning

| 问题 | 现象 | 调整措施 |
| --- | --- | --- |
| 里程计尺度标定不准 | 机器人位姿与地图位置逐渐偏离 | 标定 `ticks_per_rev` |
| 姿态漂移 | 导航过程中角度估计误差累积 | 修正 Yaw 比例 |
| 门口通行困难 | 局部规划容易保守或跳变 | 调整 `inflation_radius` 与 `robot_radius` |
| 路径跳跃 | 全局 / 局部规划切换不稳定 | 优化局部控制参数 |
| 玻璃 / 空隙误判 | 地图存在虚假开放区域 | 添加虚拟墙和人工地图修正 |
