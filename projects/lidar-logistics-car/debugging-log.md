# Debugging Log

本文件用于记录智能仓储搬运机器人的调试过程。项目调试重点不是只保存最终结果，而是保留问题、原因、排查过程和改进结论。

## 调试记录

| 问题 | 现象 | 可能原因 | 处理方式 | 结果 |
| --- | --- | --- | --- | --- |
| 玻璃 / 空隙区域导致 LiDAR 误判 | 原始地图中出现虚假可通行区域 | 激光穿透、反射或环境边界不清晰 | 基于 RViz 采点和 PGM 栅格编辑添加虚拟墙 | 修正后地图更接近真实可通行环境 |
| 可通行区域与真实障碍不一致 | 全局路径可能穿过实际障碍区域 | 原始建图结果只反映传感器可见环境 | 人工修正地图边界，重新生成 YAML 地图文件 | 提升导航安全性和可靠性 |
| 局部边界存在虚假开放区域 | 路径规划进入不可靠区域 | 地图边界不闭合或存在误识别空洞 | 使用折线 / 直线几何约束构建虚拟墙 | 局部规划更稳定 |
| 里程计尺度标定不准确 | 位姿估计与实际位置逐渐偏离 | 编码器 ticks 与实际轮距 / 轮径换算不准 | 标定 `ticks_per_rev` | 里程计输出更接近实际运动 |
| 姿态漂移 | 导航过程中角度估计误差累积 | Yaw 比例或 IMU / 里程计融合参数不合适 | 修正 Yaw 比例并继续调整融合参数 | 姿态估计稳定性改善 |
| 门口通行困难 | 门口区域路径容易跳变或通过失败 | 门口空间窄，代价地图膨胀半径和机器人半径设置偏保守 | 调整 `inflation_radius` 和 `robot_radius` | 通行能力改善，但仍需继续调参 |
| YOLO 识别不稳定 | 识别容易被背景信息干扰 | 数据集偏小、训练轮次和参数不足、背景复杂 | 与点云识别方案对比 | 最终采用点云识别作为主要定位方案 |
| 机械臂单轴动作需要校准 | 伸缩、升降、Yaw 转向需要分别验证 | 机构行程、驱动方向和限位需要确认 | 单轴调试后再进行抓取联调 | 完成桌面目标抓取验证 |

## Navigation Parameters to Track

后续每次导航调参建议记录以下参数：

| 参数 | 作用 | 调整时观察 |
| --- | --- | --- |
| `ticks_per_rev` | 编码器每圈脉冲数 / 里程计尺度 | 直线运动距离是否准确 |
| Yaw ratio | 角速度 / 姿态估计比例 | 原地旋转角度是否准确 |
| `inflation_radius` | 障碍物膨胀半径 | 门口和窄通道是否可通行 |
| `robot_radius` | 机器人半径 | 路径是否过于保守或过于贴边 |
| DWB local planner parameters | 局部轨迹采样与评分 | 是否出现振荡、跳变或绕行失败 |

## Debugging Principles

- 每次只调整少量参数，避免多个变量同时变化。
- 保留原始地图、修正地图和参数文件版本。
- 对比“修改前现象 - 修改方法 - 修改后结果”。
- 记录失败案例，因为失败案例通常比最终演示更能体现工程判断。
- 导航、识别和机械臂抓取要分别验证，再做完整任务串联。
