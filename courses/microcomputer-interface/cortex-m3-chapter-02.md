# 第 2 章 Cortex-M3 处理器知识笔记总结

## 学习目标

- 理解 Cortex-M3 的体系结构、流水线、寄存器和工作模式。
- 掌握 MSP / PSP、线程模式 / 处理模式、特权级 / 非特权级之间的关系。
- 熟悉 Cortex-M3 的统一存储器映射、位带机制、异常中断和复位流程。
- 能够把 NVIC、中断优先级、异常进入 / 退出过程和 STM32 工程调试联系起来。

## 总口诀

```text
一核两模两栈三流水，
三类特殊寄存器，
四GB映射六大区，
异常中断靠NVIC，
入栈八寄存器，
复位先取SP再取PC。
```

- 一核：Cortex-M3 内核。
- 两模：线程模式、处理模式。
- 两栈：MSP、PSP。
- 三流水：取指、译码、执行。
- 三类特殊寄存器：xPSR、中断屏蔽寄存器、CONTROL。
- 四 GB：统一地址空间。
- 六大区：Code、SRAM、外设、外部 RAM、外部设备、系统区 / PPB。
- NVIC：嵌套向量中断控制器。
- 入栈八寄存器：R0、R1、R2、R3、R12、LR、PC、xPSR。
- 复位：从 `0x00000000` 取 MSP 初始值，从 `0x00000004` 取 Reset Handler 入口。

## Cortex-M3 基本特征

Cortex-M3 是 ARM 公司设计的 32 位处理器内核，基于 ARMv7-M 架构，采用哈佛结构和三级流水线，使用 Thumb-2 指令集。它集成了 NVIC、可选 MPU、SysTick、总线接口和调试接口，适合微控制器、嵌入式控制和实时中断场景。

必须记住：

- Cortex-M3 只支持 Thumb / Thumb-2 状态，不能执行传统 ARM 指令。
- 哈佛结构表示指令总线和数据总线分离，可以边取指边访问数据。
- 三级流水线为 Fetch、Decode、Execute，即取指、译码、执行。
- PC 与流水线有关，读取 PC 时得到的值不一定等于正在执行指令的地址。
- 分支、异常和断点可能导致流水线清洗。

## 寄存器框架

### 通用寄存器

| 寄存器 | 名称 | 作用 |
| --- | --- | --- |
| R0-R12 | 通用寄存器 | 存放数据、地址和中间结果 |
| R13 | SP | 堆栈指针 |
| R14 | LR | 链接寄存器，保存返回地址或 EXC_RETURN |
| R15 | PC | 程序计数器 |

低寄存器是 R0-R7，多数 16 位 Thumb 指令可以访问。高寄存器是 R8-R12，通常需要 32 位指令或少数 16 位指令访问。

### SP：MSP 与 PSP

Cortex-M3 有两个物理堆栈指针：

- MSP：Main Stack Pointer，主堆栈指针。
- PSP：Process Stack Pointer，进程堆栈指针。

必须背：

- 复位后默认使用 MSP。
- 处理模式始终使用 MSP。
- 线程模式可以使用 MSP，也可以使用 PSP。
- PSP 常用于 RTOS 任务栈。
- SP 忽略写入值的低 2 位，因此自动 4 字节对齐。

### LR 与 PC

- 普通函数调用中，`BL function` 会把下一条指令地址保存到 LR，`BX LR` 用于返回。
- 异常处理中，LR 保存的是 EXC_RETURN，不是普通返回地址。
- PC 位 0 始终为 0；向 PC 写入数据会导致跳转，但不会自动更新 LR。

## 特殊功能寄存器

特殊功能寄存器分三类：

- 程序状态寄存器：xPSR。
- 中断屏蔽寄存器：PRIMASK、FAULTMASK、BASEPRI。
- 控制寄存器：CONTROL。

访问方式：

```asm
MRS R0, special_reg
MSR special_reg, R0
```

记忆方法：

- MRS：Special 到 Register，读特殊寄存器。
- MSR：Register 到 Special，写特殊寄存器。

## xPSR 程序状态寄存器

xPSR 由三部分组成：

| 部分 | 全称 | 作用 |
| --- | --- | --- |
| APSR | Application PSR | 运算状态标志 |
| IPSR | Interrupt PSR | 当前异常编号 |
| EPSR | Execution PSR | 执行状态 |

APSR 常见标志位：

- N：Negative，结果为负。
- Z：Zero，结果为 0。
- C：Carry，无符号进位 / 借位标志。
- V：Overflow，有符号溢出标志。
- Q：饱和标志，Cortex-M3 中通常不作为重点使用。

易混点：

- 无符号运算看 C。
- 有符号运算看 V。
- 减法时 C = 0 表示发生借位，C = 1 表示没有借位。
- IPSR = 0 表示当前没有异常；IPSR 不等于 0 表示正在处理异常。
- Cortex-M3 只支持 Thumb 状态，EPSR 中 T 位必须为 1，否则可能产生 INVSTATE 异常。

## 中断屏蔽寄存器

| 寄存器 | 作用 | 仍可响应 |
| --- | --- | --- |
| PRIMASK | 屏蔽除 NMI 和 HardFault 外的异常 | NMI、HardFault |
| FAULTMASK | 屏蔽除 NMI 外的所有异常 | NMI |
| BASEPRI | 屏蔽某优先级及更低优先级中断 | 高于门槛的中断 |

常用指令：

```asm
CPSID I   ; 关中断，PRIMASK = 1
CPSIE I   ; 开中断，PRIMASK = 0
CPSID F   ; 关异常，FAULTMASK = 1
CPSIE F   ; 开异常，FAULTMASK = 0
```

BASEPRI 的规则：优先级数值越小，实际优先级越高。若 BASEPRI 设置为优先级 2，则优先级 0、1 仍可响应，优先级 2 及更低优先级被屏蔽。

## CONTROL 控制寄存器

| 位 | 含义 | 取值 |
| --- | --- | --- |
| CONTROL[0] | 特权级控制 | 0 为特权级，1 为非特权级 |
| CONTROL[1] | 线程模式栈选择 | 0 使用 MSP，1 使用 PSP |

注意：

- CONTROL[1] 只影响线程模式。
- 处理模式永远是特权级，也永远使用 MSP。

## 工作模式与工作等级

| 模式 | 运行内容 | 特权级 | 堆栈 |
| --- | --- | --- | --- |
| Thread Mode | 普通程序 | 可特权 / 可非特权 | MSP 或 PSP |
| Handler Mode | 异常 / 中断服务程序 | 始终特权 | 只能 MSP |

三种合法组合：

- 线程模式 + 特权级。
- 线程模式 + 非特权级。
- 处理模式 + 特权级。

不存在“处理模式 + 非特权级”。

模式切换：

- 复位后：线程模式 + 特权级 + MSP。
- 发生异常：进入处理模式 + 特权级 + MSP。
- 异常返回：回到线程模式或继续处理其他异常。

## 堆栈机制

Cortex-M3 使用向下生长的满栈。

PUSH 过程：

```text
SP = SP - 4
[SP] = 数据
```

POP 过程：

```text
数据 = [SP]
SP = SP + 4
```

异常进入时，硬件自动压栈 8 个寄存器：

```text
R0, R1, R2, R3, R12, LR, PC, xPSR
```

记忆口诀：`R0-R3，R12，LR，PC，xPSR`。

异常退出时，硬件自动从栈中恢复这些寄存器。也就是说，异常进入 / 退出的基础现场保护由硬件完成。

## 存储器映射

Cortex-M3 支持 4GB 统一地址空间，范围为 `0x00000000` 到 `0xFFFFFFFF`。

| 地址范围 | 区域 | 大小 | 常见用途 |
| --- | --- | --- | --- |
| `0x00000000-0x1FFFFFFF` | Code | 512MB | 程序代码、向量表 |
| `0x20000000-0x3FFFFFFF` | SRAM | 512MB | 变量、堆、栈 |
| `0x40000000-0x5FFFFFFF` | 片上外设 | 512MB | GPIO、USART、SPI、ADC、TIM |
| `0x60000000-0x9FFFFFFF` | 外部 RAM | 1GB | 外部存储器 |
| `0xA0000000-0xDFFFFFFF` | 外部设备 | 1GB | 外部设备 |
| `0xE0000000-0xFFFFFFFF` | 系统区域 | 512MB | PPB、NVIC、SysTick、SCB |

重点理解：

- 外设寄存器也是内存地址。
- 访问外设就是访问固定地址。
- 这就是存储器映射外设。

### 端模式

大多数 Cortex-M3 微控制器使用小端模式。

- 小端：低地址存低字节。
- 大端：低地址存高字节。

例如 `0x12345678` 从 `0x4000` 开始存储，小端模式下：

| 地址 | 数据 |
| --- | --- |
| `0x4000` | `0x78` |
| `0x4001` | `0x56` |
| `0x4002` | `0x34` |
| `0x4003` | `0x12` |

## 位带 Bit-Band

位带操作可以用普通 Load / Store 指令访问单个位，作用是简化位操作、实现原子位访问，并避免传统读-改-写过程中的中断干扰。

| 类型 | 位带区 | 位带别名区 |
| --- | --- | --- |
| SRAM | `0x20000000-0x200FFFFF` | `0x22000000-0x23FFFFFF` |
| 外设 | `0x40000000-0x400FFFFF` | `0x42000000-0x43FFFFFF` |

映射关系：

```text
位带区中的 1 bit -> 别名区中的 1 个 32 位字
```

SRAM 位带公式：

```text
AliasAddr = 0x22000000 + (Addr - 0x20000000) * 32 + n * 4
```

外设位带公式：

```text
AliasAddr = 0x42000000 + (Addr - 0x40000000) * 32 + n * 4
```

其中 `Addr` 是目标位所在字节地址，`n` 是目标位在该字节中的位号，范围为 0-7。

公式记忆：`别名基址 + 字节偏移 * 32 + 位号 * 4`。

## 异常与中断

异常是打断程序正常执行流程的事件，中断是异常的一类。

常见异常包括：

- Reset。
- NMI。
- HardFault。
- Fault。
- 外部中断。
- SVCall。
- PendSV。
- SysTick。

中断过程：

```text
中断请求 -> 中断响应 -> 执行 ISR -> 中断返回
```

中断和子程序调用的区别：

| 对比项 | 中断 | 子程序调用 |
| --- | --- | --- |
| 产生时刻 | 随机，由事件触发 | 程序预先安排 |
| 入口来源 | 硬件和向量表确定 | 程序指令指定 |
| 现场保护 | 硬件自动压栈并可能需要软件补充保护 | 主要保存返回地址 |
| 来源 | 硬件或软件事件 | 调用指令 |

查询与中断：

- 查询：CPU 周期性检查设备状态，是“我去问”。
- 中断：事件发生时通知 CPU，是“你来叫”。

## NVIC 与优先级

NVIC 全称 Nested Vectored Interrupt Controller，即嵌套向量中断控制器。

核心功能：

- 支持中断嵌套。
- 支持向量中断。
- 支持动态优先级配置。
- 支持中断屏蔽。
- 支持低延迟中断响应。
- 支持咬尾和迟来异常。

常见异常编号：

| 异常号 | 名称 |
| --- | --- |
| 0 | 无异常 |
| 1 | Reset |
| 2 | NMI |
| 3 | HardFault |
| 4 | Memory Management Fault |
| 5 | Bus Fault |
| 6 | Usage Fault |
| 11 | SVCall |
| 14 | PendSV |
| 15 | SysTick |
| 16 及以上 | 外部中断 |

固定高优先级异常：

- Reset：优先级 -3，最高。
- NMI：优先级 -2。
- HardFault：优先级 -1。

优先级规则：

- 优先级数值越小，实际优先级越高。
- 若软件优先级相同，异常编号越小，硬件优先级越高。
- Reset、NMI、HardFault 的优先级固定，不受软件设置影响。

### 抢占优先级与副优先级

- 抢占优先级决定能不能打断正在执行的中断。
- 副优先级决定抢占优先级相同时谁先响应。
- 副优先级不能产生抢占。

PRIGROUP 位于 AIRCR 寄存器的 `[10:8]` 位，用于决定优先级字段中多少位给抢占优先级、多少位给副优先级。STM32 常见情况是使用高 4 位作为有效优先级位。

## 异常进入与异常退出

异常进入硬件自动步骤：

1. 硬件压栈。
2. 读取向量表。
3. 更新 PC。
4. 更新 LR 为 EXC_RETURN。
5. 加载流水线。
6. 执行 ISR。

向量地址计算：

```text
向量地址 = 向量表基址 + 异常号 * 4
```

例如外部 IRQ0 的异常号为 16，若向量表基址为 `0x00000000`，则向量地址为：

```text
0x00000000 + 16 * 4 = 0x00000040
```

EXC_RETURN 不是普通地址，而是异常返回标志值。常见值：

| EXC_RETURN | 含义 |
| --- | --- |
| `0xFFFFFFF1` | 返回处理模式，使用 MSP |
| `0xFFFFFFF9` | 返回线程模式，使用 MSP |
| `0xFFFFFFFD` | 返回线程模式，使用 PSP |

异常返回常见触发方式：

```asm
BX LR
```

当 LR 中保存的是 EXC_RETURN 时，执行 `BX LR` 会触发异常返回流程。

## 中断嵌套、咬尾和迟来异常

中断嵌套：一个 ISR 执行过程中，如果来了抢占优先级更高的异常，处理器会打断当前 ISR，转去执行更高优先级 ISR。

典型过程：

```text
线程程序 -> 中断1 -> 高优先级中断2 -> 返回中断1 -> 返回线程程序
```

| 机制 | 发生时机 | 作用 | 记忆 |
| --- | --- | --- | --- |
| 咬尾 Tail-Chaining | 一个 ISR 结束时已有下一个挂起中断 | 省去完整出栈 / 入栈 | 看尾巴 |
| 迟来异常 Late Arrival | 异常进入压栈阶段来了更高优先级异常 | 更快响应高优先级异常 | 看入口 |

## Fault 类异常

Cortex-M3 中常见 Fault：

- Bus Fault：总线 Fault。
- Memory Management Fault：存储器管理 Fault。
- Usage Fault：用法 Fault。
- Hard Fault：硬 Fault。

Fault 状态寄存器会保持状态，直到手动清除。清除方式通常是写 1 清除。调试 HardFault 时要注意区分新旧 Fault 状态，处理后应清除相关标志。

## 复位序列

复位让处理器进入确定的初始状态。Cortex-M3 程序映像从 `0x00000000` 开始，程序映像开始处是向量表。

复位后启动流程：

1. 从 `0x00000000` 读取 MSP 初始值。
2. 从 `0x00000004` 读取复位向量，也就是 Reset Handler 入口地址。
3. 从 Reset Handler 开始取第一条指令执行。

向量表中异常处理函数地址最低位通常置 1，表示 Thumb 指令状态。例如实际入口地址为 `0x00000100`，向量表中可能写入 `0x00000101`。

## 必背对比表

### MSP vs PSP

| 项目 | MSP | PSP |
| --- | --- | --- |
| 全称 | Main Stack Pointer | Process Stack Pointer |
| 中文 | 主堆栈指针 | 进程堆栈指针 |
| 复位后默认 | 是 | 否 |
| 处理模式 | 使用 MSP | 不使用 |
| 线程模式 | 可使用 | 可使用 |
| 典型用途 | 中断、异常、裸机程序 | RTOS 任务栈 |

### Thread Mode vs Handler Mode

| 项目 | 线程模式 | 处理模式 |
| --- | --- | --- |
| 英文 | Thread Mode | Handler Mode |
| 运行内容 | 普通程序 | 异常 / 中断服务程序 |
| 特权级 | 可特权 / 可非特权 | 始终特权 |
| 堆栈 | MSP 或 PSP | 只能 MSP |
| 进入方式 | 复位后、异常返回 | 异常发生 |

### C 标志 vs V 标志

| 标志 | 用于判断 | 含义 |
| --- | --- | --- |
| C | 无符号运算 | 进位 / 借位 |
| V | 有符号运算 | 溢出 |

口诀：无符号看 C，有符号看 V。

## 简答题模板

### 简述 Cortex-M3 特点

Cortex-M3 是 ARM 公司设计的 32 位处理器内核，采用 ARMv7-M 架构，基于哈佛结构，具有三级流水线，使用 Thumb-2 指令集。它集成了 NVIC、MPU、SysTick、总线接口和调试接口，支持低延迟中断、多级中断嵌套、位带操作和统一的 4GB 存储器映射。

### Cortex-M3 有哪些工作模式？

Cortex-M3 有线程模式和处理模式。线程模式用于普通程序执行，可以是特权级或非特权级；处理模式用于异常和中断处理，始终为特权级。

### Cortex-M3 堆栈有哪些特点？

Cortex-M3 采用向下生长的满栈，堆栈按 32 位字操作。PUSH 时 SP 先减 4 再存数据，POP 时先读数据再 SP 加 4。Cortex-M3 有两个堆栈指针 MSP 和 PSP，复位后默认使用 MSP，异常处理始终使用 MSP，线程模式可使用 MSP 或 PSP。

### 异常进入时压栈哪些寄存器？

异常进入时，硬件自动将 R0、R1、R2、R3、R12、LR、PC 和 xPSR 压入当前使用的堆栈。

### NVIC 有什么作用？

NVIC 是 Cortex-M3 内置的嵌套向量中断控制器，用于管理异常和中断。它支持中断嵌套、向量中断、动态优先级配置、中断屏蔽、低延迟响应、咬尾和迟来异常等机制。

### 位带操作的原理和作用是什么？

位带操作是将位带区中的每一位映射到别名区中的一个 32 位字，通过访问别名区地址即可实现对原始位的读写。它可以简化单个位操作，实现原子访问，避免传统读-改-写过程中的干扰。

### 复位后 Cortex-M3 如何启动？

复位后，Cortex-M3 首先从地址 `0x00000000` 读取 MSP 初始值，然后从地址 `0x00000004` 读取复位向量，即 Reset Handler 入口地址，最后从该地址开始取指执行程序。

## 与项目的联系

- STM32 的外设寄存器访问本质上依赖 Cortex-M3 的存储器映射外设思想。
- GPIO、EXTI、Timer、USART 等外设调试都绕不开 NVIC、中断优先级和异常响应机制。
- SysTick 可用于周期任务、系统节拍和基础调度实验。
- HardFault、BusFault、UsageFault 的理解有助于定位野指针、非法访问、栈溢出和错误跳转。
- MSP / PSP 与异常压栈机制是学习 RTOS 任务切换、PendSV 和上下文保存的基础。

- R0-R12 通用，R13 = SP，R14 = LR，R15 = PC。
- SP 有 MSP 和 PSP。
- 复位后默认 MSP，异常使用 MSP，线程模式可用 PSP。
- xPSR = APSR + IPSR + EPSR。
- APSR 标志：N、Z、C、V。
- 无符号看 C，有符号看 V。
- PRIMASK 屏蔽除 NMI 和 HardFault 外的异常。
- FAULTMASK 只允许 NMI。
- BASEPRI 屏蔽某优先级及更低优先级。
- CONTROL[0] 控制特权级，CONTROL[1] 控制线程模式用 MSP / PSP。

- 堆栈：向下生长满栈，PUSH 先减 4，POP 后加 4。
- 异常入栈八个：R0、R1、R2、R3、R12、LR、PC、xPSR。
- 4GB 地址空间：Code、SRAM、外设、外部 RAM、外部设备、系统区。
- 位带公式：别名基址 + 字节偏移 * 32 + 位号 * 4。
- 异常编号 1-15 为系统异常，16 及以上为外部中断。
- Reset = -3，NMI = -2，HardFault = -1。
- 优先级数值越小，优先级越高。
- 抢占优先级决定能否打断，副优先级决定同组先后。
- EXC_RETURN 不是地址，是异常返回标志。
- `0xFFFFFFF9` 返回线程模式并使用 MSP。
- `0xFFFFFFFD` 返回线程模式并使用 PSP。
- 咬尾发生在 ISR 结束时，迟来异常发生在异常进入压栈时。
- 复位先读 `0x00000000` 的 MSP，再读 `0x00000004` 的 Reset Handler。
